HTML Basics

HTML (HyperText Markup Language) is the standard language for web pages.

It defines the structure of a webpage using elements.

  • Headings (h1-h6)

    Headings are used to define the structure of your content, from the largest (h1) to the smallest (h6).

    <h1>Heading 1</h1>

    <h2>Heading 2</h2>

    <h3>Heading 3</h3>

    <h4>Heading 4</h4>

    <h5>Heading 5</h5>
    <h6>Heading 6</h6>
  • Paragraphs (p)

    Paragraphs are used to define blocks of text in a webpage.

    <p>This is a paragraph</p>

    Output:

    This is a paragraph

  • Lists (ul, ol, li)

    Lists can be either ordered (ol) or unordered (ul). Each item is defined by the list tag.

    <ul><li>Item 1</li><li>Item 2</li></ul>

    Output:

    • Item 1
    • Item 2

    <ol><li>First item</li><li>Second item</li></ol>

    Output:

    1. First item
    2. Second item

CSS Styling

CSS (Cascading Style Sheets) styles HTML elements.

It controls layout, colors, fonts, and more.

body { background-color: lightgray; }
  • Selectors

    A CSS selector is used to select an HTML element and apply styles to it.

    p { color: blue; }

    Output: This text is blue.

  • Box Model

    The CSS box model is a box that wraps around every HTML element, consisting of margins, borders, padding, and the content itself.

    div { padding: 10px; border: 2px solid black; }

    Output:

    This is a div with padding and border.

  • Flexbox & Grid

    Flexbox and Grid are layout models in CSS used to arrange elements in rows or columns.

    div { display: flex; justify-content: center; }

    Output:

    This text is centered.

Data Types

JavaScript has several data types that are essential for working with values in your applications.

The data types can be divided into two categories: Primitive data types and Objects.

  • Boolean

    The Boolean data type has two possible values: true and false.

    let isActive = true;

    Output: The value of isActive is true, meaning the variable is active.

  • Null

    The null data type represents the intentional absence of any object value.

    let user = null;

    Output: The value of user is null, indicating that the user does not exist or is undefined.

  • Undefined

    The undefined data type represents a variable that has been declared but not assigned a value.

    let name;

    Output: The value of name is undefined because it has been declared but not assigned any value.

  • Number

    The Number data type represents both integers and floating-point numbers.

    let age = 25;

    Output: The value of age is 25, which is a number representing someone's age.

    let pi = 3.14159;

    Output: The value of pi is 3.14159, which is a floating-point number.

  • String

    The String data type represents a sequence of characters enclosed in single, double, or backticks (for template literals).

    let greeting = "Hello, world!";

    Output: The value of greeting is "Hello, world!", which is a string of characters.

  • Symbol (ES6)

    The Symbol data type is a unique and immutable data type introduced in ES6 that can be used to create unique identifiers.

    let uniqueId = Symbol('id');

    Output: The value of uniqueId is a unique symbol, which is not equal to any other symbol or string.

  • Object

    The Object data type is a collection of properties, where each property has a key (string) and a value (any data type).

    let person = { name: "John", age: 30, isActive: true };

    Output: The value of person is an object with three properties: name, age, and isActive.

Loops

Loops are used to execute a block of code repeatedly as long as a specified condition is met.

JavaScript has several types of loops, each serving different use cases.

  • For Loop

    The for loop is typically used when you know in advance how many times you want to execute a statement or a block of code.

    for (let i = 0; i < 5; i++) {
        console.log(i);
    }

    Output: The loop will print numbers from 0 to 4:

    0
    1
    2
    3
    4

  • While Loop

    The while loop repeats as long as the specified condition is true.

    let i = 0;
    while (i < 5) {
        console.log(i);
        i++;
    }

    Output: The loop will print numbers from 0 to 4:

    0
    1
    2
    3
    4

  • Do...While Loop

    The do...while loop executes the block of code once before checking the condition, and then it repeats as long as the condition is true.

    let i = 0;
    do {
        console.log(i);
        i++;
    } while (i < 5);

    Output: The loop will print numbers from 0 to 4:

    0
    1
    2
    3
    4

  • For...In Loop

    The for...in loop is used to iterate over the properties of an object.

    let person = { name: "John", age: 30, job: "Developer" };
    for (let key in person) {
        console.log(key + ": " + person[key]);
    }

    Output: The loop will print each property of the object:

    name: John
    age: 30
    job: Developer

  • For...Of Loop

    The for...of loop is used to iterate over iterable objects such as arrays.

    let numbers = [10, 20, 30, 40];
    for (let num of numbers) {
        console.log(num);
    }

    Output: The loop will print the values in the array:

    10
    20
    30
    40

Responsive Design

Responsive design ensures websites work on all screen sizes.

Techniques include flexible layouts, media queries, and scalable assets.

  • Media Queries

    Media queries allow different styles to be applied based on conditions like screen width, height, orientation, etc.

    @media (max-width: 600px) { body { font-size: 14px; } }

    Output: On screens smaller than 600px, the body font-size will be reduced to 14px. This is a simple example of applying responsive styles using media queries.

  • Viewport Units

    Viewport units allow you to size elements relative to the viewport's size. There are four types of viewport units: vw (viewport width), vh (viewport height), vmin, and vmax.

    h1 { font-size: 10vw; }

    Output: The font-size of the `

    ` element will adjust based on 10% of the viewport's width. As the viewport size changes, the text size will scale accordingly.

  • Mobile-First Design

    Mobile-first design is a strategy where you start by designing for smaller screens and progressively add styles for larger screens.

    @media (min-width: 768px) { body { font-size: 16px; } }

    Output: Initially, the design will be optimized for mobile devices. As the screen width increases to 768px or more, the body font-size will be increased to 16px for larger devices like tablets and desktops.

Best Practices

Best coding practices improve performance and maintainability.

Follow industry standards to write clean, efficient code.

  • Code Readability

    Writing code that is easy to read and understand is crucial for team collaboration and maintenance. Use descriptive variable names and comments where necessary.

    const calculateArea = (width, height) => width * height;

    Explanation: The function name calculateArea clearly describes its purpose, making it easy to understand at a glance.

    Output: This code calculates the area of a rectangle when called with appropriate arguments.

  • Performance Optimization

    Optimize code to improve performance, particularly for applications with large data sets or intensive operations. Consider time complexity and memory usage.

    let numbers = [1, 2, 3, 4, 5];
    let sum = numbers.reduce((acc, num) => acc + num, 0);

    Explanation: The reduce() method is an efficient way to sum elements in an array.

    Output: The code calculates the sum of all numbers in the array.
